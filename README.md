# 自动化设备客户协议评估成本核算系统（首版）

> 目标：构建一个**纯网站应用**，优先支持视觉评估，同时为机构/电气扩展预留接口。

## 规划（先规划后开发）

### 1) 系统范围与角色
- 使用角色：设计工程师、产品经理
- 当前范围：视觉方案评估、标准件维护、品牌偏离建议、报告导出
- 预留扩展：机构/电气评估模块、专业文档解析服务

### 2) 模块拆分
- 客户资料解读模块：文件导入、关键参数提取、原档备注
- 标准件管理模块：型号/品牌/价格/精度能力/交期维护
- 智能核算模块：检测精度与产品参数关联计算、成本估算
- 品牌偏离模块：偏离识别 + 替代品牌优点指引
- 报告输出模块：Markdown / JSON 报告导出

### 3) 技术方案（无第三方依赖）
- 前端：原生 HTML + CSS + JavaScript
- 存储：`localStorage`
- 服务：Node.js 内置 `http` 静态服务（无 npm 依赖）
- 解析策略：
  - 首版原生支持：TXT / CSV / JSON
  - 预留接口支持：PDF / Word / Excel（通过 `/api/parse/reserved`）

### 4) 数据模型（核心）
- 标准件：`category`, `name`, `model`, `brand`, `price`, `specPrecisionUm`, `leadTimeDays`
- 客户文档：`fileName`, `ext`, `parseLevel`, `highlights`, `note`
- 评估结果：输入参数、推荐清单、品牌偏离建议、成本汇总

## 已实现功能

1. 文档导入与解析（含扩展位）
2. 标准件增删查与持久化
3. 视觉精度智能核算（基于规则引擎）
4. 品牌偏离推荐与优点提示
5. 报告预览与导出（Markdown / JSON）

## 运行方式

```powershell
cd "e:\协议评估成本核算系统"
node server.js
```

浏览器打开：`http://localhost:3000`

## 推荐开发运维命令（稳定）

```powershell
cd "e:\协议评估成本核算系统"
npm run dev:start
npm run dev:status
npm run dev:stop
```

- `dev:start`：后台稳定启动，自动写入PID与日志。
- `dev:status`：查看监听状态与最近日志。
- `dev:stop`：精准停止当前项目服务。
