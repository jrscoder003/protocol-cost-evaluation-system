# 更新记录：v0.11.9 上传500防复发修复

- 日期：2026-03-04
- 目标：解决“协议上传失败（status code 500）”的复发问题，并提供防复发保护。

## 一、问题分析

### 现象
- 前端上传弹窗提示：`上传失败: Request failed with status code 500`。

### 高风险触发点
1. 上传文件名字符在编码转换后出现非法字符，导致保存或归档异常。
2. `req.files` 异常为空时直接访问 `.length`，可能触发服务端错误。
3. 归档复制失败时直接中断主流程，用户侧表现为 500。

## 二、修复措施

文件：`backend/routes/agreement.ts`
- 新增 `decodeOriginalName(rawName)`：文件名解码失败时回退原值。
- 新增 `sanitizeFileName(input)`：剔除 Windows 非法字符（`\\/:*?"<>|`）。
- 上传落盘文件名改为“解码+安全化”后再写入。
- `req.files` 改为安全兜底：`(req.files as Express.Multer.File[]) || []`。
- 归档复制改为 `try/catch`：失败仅记日志，不阻断上传主流程。
- 列表接口数组处理加强类型声明，避免映射阶段类型歧义。

## 三、验证结果
- 使用真实测试目录样本上传：成功。
- 接口返回：`Files uploaded successfully`。
- 最新记录包含 `archive_url`，可用于“查看直开原文”。

## 四、结论
- 本次已对上传链路做“编码/文件名/空请求/归档失败”四层防护。
- 能有效规避此前反复出现的 500 上传失败问题。
