# 更新记录：v0.11.14 关键词映射缩影与导出500修复

- 日期：2026-03-05
- 目标：修复“下载当前关键词库”失败，并把关键词映射关系按小白模板可视化展示。

## 一、问题与根因

1. “下载当前关键词库”点击后前端提示 `Request failed with status code 500`。
2. 根因：`backend/routes/keyword.ts` 的导出接口在 ESM 运行环境中使用了 `require('xlsx')`，触发运行时异常。
3. 页面虽然有关键词与绑定功能，但关键词页缺少“模板视角”的映射缩影，现场工程师阅读成本高。

## 二、本次实现

### 1) 导出500修复 + 全量导出结构
- 文件：`backend/routes/keyword.ts`
- 修复点：移除 `require('xlsx')`，统一使用顶部已导入的 `XLSX`。
- 导出结构改为“关键词+绑定逐行展开”：
  - 若有绑定：按每条绑定导出一行。
  - 若无绑定：按 `ALL/ALL` 兜底导出一行。
- 导出列与小白模板一致：
  - `机型`
  - `工位`
  - `解析关键项`（中文）
  - `关键词引词`
  - `关联项`
  - `项目实际解析的内容简介（备注）`

### 2) 关键词页映射缩影展示
- 文件：`src/views/LibraryView.vue`
- 新增“映射缩影（按导出模板展示）”表，直接显示模板六列。
- 主关键词表优化：
  - `关键项` 改为中文显示（`productSize -> 产品尺寸` 等）。
  - `绑定范围`无绑定时显示 `ALL/ALL（默认）`。
- 标签切换优化：进入“关键词索引”时同步加载 `bindingList`，保证映射即时显示。

### 3) 下载按钮语义修正
- 文件：`src/views/LibraryView.vue`
- `下载导入模板` 按钮固定下载示例模板（csv）。
- `下载当前关键词库` 按钮下载当前库真实数据（xlsx）。
- 前端下载函数增加对 `content-disposition` 文件名解析与 `blob` 兼容。

## 三、验证要点

1. 点击“下载当前关键词库”不再出现 500，浏览器可下载 `.xlsx` 文件。
2. 下载内容包含库内已有关键词与绑定（无绑定项自动按 `ALL/ALL` 展开）。
3. 关键词维护页可直接看到“映射缩影”六列信息。
4. 切换到“关键词索引”页时，“绑定范围”不再出现大面积空白误导。

## 四、后续建议

- 支持在映射缩影表中一键导出当前筛选结果。
- 增加“仅显示未绑定项”过滤，便于补齐绑定。
- 在映射缩影中增加“来源”标识（手工/导入/默认种子），便于治理关键词质量。
