# 更新记录：v0.11.3 P0 工位解析优化

- 日期：2026-03-04
- 范围：前端解析逻辑（`public/app.js`）与规则文档（`docs/`）
- 目标：修复工位解析不稳定问题，提供可解释命中证据，建立每次优化留痕机制。

## 一、变更内容

### 1) 解析规则扩展（P0）
- 新增多规则工位抽取管线：`parseStationIntelligence(...)`
- 覆盖表达：
  - `1工位 / 2站 / 第3工位`
  - `工位1 / 工站-02`
  - `Station 3 / STN-04 / STA_05`
  - `S01 / S-2`
  - `3#站`
  - `一站 / 第二工位 / 十工位`

### 2) 统一编号归一化
- `normalizeStationNo(...)` 支持阿拉伯数字与中文数字。
- 新增 `chineseNumeralToNumber(...)`，将 `一/二/十/十一/两` 等转换为数字。
- 输出统一格式：`N站`。

### 3) 命中证据输出
- 预览逻辑改为输出证据链：规则名、来源、置信度、命中片段。
- 页面在“解析验证”区域新增“命中证据”表格（嵌入现有预览面板）。

### 4) 代码可维护性提升
- 关键解析函数补充注释：
  - 解析主流程注释
  - 中文数字转换注释
  - 置信度与命中片段生成说明

## 三、影响范围
- 视觉方案报告导入解析（工位识别更稳，编号更统一）

- 下游依赖 `inferStations` / `inferStationDetails` 的逻辑会获得更一致的工位编号

## 四、验证方式
1. 选择包含工位信息的文件，点击“解析并展示细节”。
2. 确认：
   - “识别工位”输出为 `N站`。
   - “命中证据”表格可展示规则、来源、片段。
   - 同一文件重复解析结果一致。
3. 进行语法校验：`node --check public/app.js`。
