# 更新记录：v0.11.7 访问中断与运维脚本加固

- 日期：2026-03-04
- 目标：解决 `localhost:3004/library` 反复无法访问问题，提供稳定的一键启停方案。

## 一、问题现象
- 浏览器访问 `http://localhost:3004/library` 出现 `ERR_CONNECTION_REFUSED`。
- 后端 `3006` 可用，但前端 `3004` 未常驻或被端口冲突影响。

## 二、根因分析
1. 前端 `vite` 进程在部分场景下未持续运行（终端中断/端口冲突后退出）。
2. 项目原有 `scripts/start.ps1`/`stop.ps1`/`status.ps1` 仍是旧版单服务（`server.js:3000`）逻辑，不适配当前 `vite(3004)+backend(3006)` 架构。

## 三、修复内容

### 1) 运维脚本重构
- `scripts/start.ps1`
  - 自动清理 `3004/3006` 冲突占用。
  - 启动前端与后端两个进程（隐藏窗口）。
  - 记录 PID 与日志到 `.runtime/`。

- `scripts/stop.ps1`
  - 按 PID 停止前后端。
  - 补充端口兜底清理，避免残留监听。

- `scripts/status.ps1`
  - 显示前后端独立状态、PID、URL。
  - 输出最近日志尾部，便于排查。

### 2) npm 命令增强
- `package.json` 新增：
  - `dev:start`
  - `dev:stop`
  - `dev:status`

## 四、验证结果
- 后端：`http://localhost:3006/api/health` 返回 `{"ok": true}`。
- 前端：`http://localhost:3004/` 返回首页内容（系统标题与模块卡片）。
- 脚本文件语法检查：无错误。

## 五、使用建议
- 日常启动：`npm run dev:start`
- 日常停止：`npm run dev:stop`
- 状态检查：`npm run dev:status`
- 如再次异常，先执行 `dev:stop` 再 `dev:start`，不要混用多终端重复启动。
