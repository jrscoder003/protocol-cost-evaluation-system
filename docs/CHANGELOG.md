# 开发更新日志（版本化）

> 规则：每次完成开发/优化后，都新增一条版本记录。

## [v0.11.14] - 2026-03-05
### 修复
- 修复“下载当前关键词库”报 `500`：`/api/keywords/export-template` 在 ESM 环境中误用 `require('xlsx')` 导致导出异常，现改为统一使用已导入的 `XLSX`。

### 优化
- 关键词维护页新增“映射缩影（按导出模板展示）”表，按小白模板字段直观展示：`机型/工位/解析关键项/关键词引词/关联项/项目实际解析的内容简介（备注）`。
- 关键词主表“关键项”改为中文展示（如 `productSize -> 产品尺寸`），降低阅读门槛。
- “绑定范围”无显式绑定时改为 `ALL/ALL（默认）`，避免“未绑定”造成理解偏差。
- 标签切换到“关键词索引”时同步加载绑定数据，确保映射展示即时可见。
- 导出文件改为“按关键词+绑定逐行展开”，完整保存库内已有关键词与绑定关系。

### 交互
- 区分下载行为：
	- `下载当前关键词库`：下载当前库内真实数据（xlsx）。
	- `下载导入模板`：下载示例模板（csv）。

## [v0.11.10] - 2026-03-05
### 修复
- 修复“上传失败 status code 500”复发根因：前端单独启动时，`Vite` 代理在后端离线场景返回 `500`，现已通过启动链路与提示链路双重修复。
- 修复运维状态误判问题：`status.ps1` 之前按临时父进程 PID 判活，可能出现“服务在线但显示离线”。

### 优化
- 调整默认开发命令：`npm run dev` 改为同时启动前端与后端，减少“只起前端导致上传失败”的操作风险。
- `scripts/start.ps1` 改为记录端口真实监听进程 PID，避免 PID 漂移导致的误判。
- `scripts/status.ps1` 改为优先依据端口监听状态判定服务是否在线，并自动纠正 PID 文件。
- `src/services/api.ts` 新增统一响应错误归一化：后端离线/代理拒绝连接时，前端提示“后端服务不可用（请确认3006端口）”。

### 验证
- 执行 `npm run dev:stop; npm run dev:start; npm run dev:status`，前后端均显示 running。
- 使用 `scripts/test-upload-folder.mjs` 真实样本复测上传成功，返回 `Files uploaded successfully`。
- 新增记录可写入并读取，最近记录 ID 正常递增。

## [v0.5.0] - 2026-03-02
### 新增
- 视觉导入新增“文件夹导入”能力（`webkitdirectory`），支持离散文件+文件夹联合解析。
- 新增“重点评估内容描述”输出：自动提取需求并给出视觉方案计算说明。
- 维护页新增“参考文档加载”能力，并接入与导入资料的匹配比对。
- 维护页新增“产品类型能力维护”：产品类型、工位内容、工位能力、工位节拍。

### 优化
- 收紧整体布局宽度与栅格，减少左右留白，提升大屏利用率。
- 解析结果显示文件相对路径，支持文件夹场景追溯。
- 仪表盘增加参考文档数、产品类型能力数等运营指标。

### 计算说明
- 新增自主判定逻辑：
	- 目标精度/节拍推断；
	- 视觉硬件配置计算（建议分辨能力、像素能力估算）；
	- 检测能力匹配（特征匹配率 + 工位能力匹配率 + 综合评分）；
	- 软件功能匹配建议（OCR/缺陷/测量等）。

### 修复
- 导入区去重并统一处理，避免重复文件导致结果干扰。

## [v0.4.0] - 2026-03-02
### 新增
- 科技感后台UI重构（深色工业风、侧栏菜单、驾驶舱、面包屑、返回上个模块）。
- 菜单路由页体验：单页模块切换、Hash路由、导航状态同步。
- 视觉评估模块子标签：导入/维护/评估/报告。
- Root管理页占位与口令验证交互（示例口令 `root123`）。

### 修复
- 解决 `localhost:3000` 反复 `ERR_CONNECTION_REFUSED` 的根因：并非项目服务在运行。
- 新增稳定运维脚本，避免编辑后服务未常驻导致页面无法访问。

### 运维
- 新增 `scripts/start.ps1`：PID+日志模式稳定启动（端口监听检查）。
- 新增 `scripts/stop.ps1`：按PID精准停止服务。
- 新增 `scripts/status.ps1`：查看运行状态与最近日志。
- 新增 npm 命令：`dev:start`、`dev:stop`、`dev:status`。
